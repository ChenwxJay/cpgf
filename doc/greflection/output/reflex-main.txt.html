cppreflex
Features:

1, Can reflect a lot of information.

  • Member and static fields with any data type.
  • Global fields (global variable) with any data type.
  • Properties (fields with getter and setter). Can be either member or global.
  • Member and static functions with any parameters, return value, and calling convention.
  • Global functions.
  • Enumerators.
  • Classes. Inner class can be reflected to outter meta class. Top level classes can be reflected to global meta class.
  • Object contructors.
  • Operators. All overloadable operators, except new and delete, are supported. The functional of overloaded new and delete are already provided by meta class.
  • Annotations. Any annotations, in key-value pairs, can be reflected. Value can be any type.

2, We can do a lot with the reflection system.

  • Create object from class name, without knowing the class type.
  • Discover the class hierarchy.
  • Get or set any object fields directly or via getter and setter (property) at runtime.
  • Invoke any functions at runtime. Function parameter and result types can be discovered at runtime.
  • Invoke any overloaded operators at runtime.
  • Discover enumerator types at runtime.
  • More. Waiting for you to discover.

3, Accessing fields and properties, calling methods, are truely runtime behavior, no template parameters are needed. All parameters and return values are passed via GVariant.

4, Automatically type conversion when getting/setting fields, invoking methods, etc.

3, Support both intrusive and non-intrusive meta data. Reflection code can be put inside the class declaration (intrusive), or put anywhere outside of the class (non-intrusive).

4, No preprocessor required. Only using templates and macros.

5, Written in standard and portable C++, compiled on VC 2008 and GCC 4.5.2.

6, Easy to build the meta data. No ugly macros. Some macros are used in quite function style.

7, Support multiple inheritance.

8, Declarative programming with annotations
